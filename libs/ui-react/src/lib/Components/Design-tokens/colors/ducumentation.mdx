# Colors

Our color system relies on semantic tokens, simplifying design decisions and enabling dark theme support.

## Taxonomy

All colors tokens follow the structure : property-variant-state

<img src={taxonomy} alt="A description of the image" />

Exemple :

- `background-accent-hover`
- `border-muted-pressed`

## Element

<Markdown>
  {`
| Element | Definition | Example |
| ------- | ---------- | ------- |
| property | Broad category of usage for the color | background, border, text |
| variant  | Property by specifying a contextual usage | base, muted, interactive |
| state    | Interaction-based adjustment of the token | default, hover, pressed |
`}

</Markdown>

## Color token properties

<img src={tokenProperty} alt="A description of the image" />

The property defines the broad categories of usage for the colors.
Tokens property are divided in 5 categories.

The table below lists our color tokens properties, showing their typical usage to help maintain visual consistency across the interface.

<Markdown>
  {`
| Properties | Usage |
| ---------- | ----- |
| background | Appears behind content, serving as the visual foundation for screens and components |
| border | Used for borders around elements, helping define structure and separate content |
| text | Applied to textual content and icons, including headings, paragraphs, and labels |
| crypto | Covers all graphical elements related to cryptocurrencies, such as logos |
| discover | Cover all graphical element related to discover and provider element |
`}
</Markdown>

## Tokens variants

Variant property refines Property by specifying a contextual usage.
Token variants are divided in this following categories.

<img src={tokenVariant} alt="A description of the image" />

The table below lists our color token variants, showing their typical usage to help maintain visual consistency across the interface.

<Markdown>
  {`
| Variants | Usage |
| -------- | ----- |
| base | Color by default |
| constant (white,black,grey) | Constant colors in light and dark mode |
| muted | Default less attention grabbing colors |
| accent | Draw attention, highlight, key elements |
| status | Success, error, warning colors |
| interactive | Colors for element required user interaction |
| active | Colors used to indicate an element in currently selected |
| focus | Colors used to indicate an element selected by keyboard navigation |
| on-accent | Colors used on accent colors |
| on-interactive | Colors used on interactive colors |
| on-status | Colors used on status colors |
`}
</Markdown>

## Tokens states

The state represent an interaction-bases adjustment of the token.
Token states are divided in 4 categories.

<img src={tokenState} alt="A description of the image" />

<Markdown>
  {`
| States | Usage |
| ------ | ----- |
| default | Default color without any user interaction |
| hover | Colors applied when a user move their cursor over an interactive component |
| pressed | Colors displayed when the element is actively clicked or tapped |
| disabled | Colors used when an element is inactive or non-interactable |
`}
</Markdown>
