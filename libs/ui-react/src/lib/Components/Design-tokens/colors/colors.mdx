Our color system relies on semantic tokens, simplifying design decisions and enabling dark theme support.

## Color Taxonomy

All colors tokens follow the structure: property-variant-state

<img src={taxonomy} alt="Color taxonomy diagram" />

Example:

- `background-accent-hover`
- `border-muted-pressed`

## âš¡ Element Structure

<Markdown>
  {`
| Element | Definition | Example |
| ------- | ---------- | ------- |
| property | Broad category of usage for the color | background, border, text |
| variant  | Property by specifying a contextual usage | base, muted, interactive |
| state    | Interaction-based adjustment of the token | default, hover, pressed |
`}
</Markdown>

## ðŸ“Š Color Token Properties

<img src={tokenProperty} alt="Token properties diagram" />

The property defines the broad categories of usage for the colors.
Tokens property are divided in 5 categories.

Here's how our color tokens properties help maintain visual consistency:

<Markdown>
  {`
| Properties | Usage |
| ---------- | ----- |
| background | Appears behind content, serving as the visual foundation |
| border | Used for borders around elements and structure |
| text | Applied to textual content and icons |
| crypto | Covers cryptocurrency-related elements |
| discover | Cover discover and provider elements |
`}
</Markdown>

## Token Variants

Variant property refines Property by specifying a contextual usage.
Token variants are divided into these categories:

<img src={tokenVariant} alt="Token variants diagram" />

<Markdown>
  {`
| Variants | Usage |
| -------- | ----- |
| base | Color by default |
| constant | Constant colors in light and dark mode |
| muted | Less attention grabbing colors |
| accent | Draw attention, highlight elements |
| status | Success, error, warning colors |
| interactive | Colors for interactive elements |
| active | Currently selected elements |
| focus | Keyboard navigation selected elements |
| on-accent | Colors used on accent colors |
| on-interactive | Colors used on interactive colors |
| on-status | Colors used on status colors |
`}
</Markdown>
