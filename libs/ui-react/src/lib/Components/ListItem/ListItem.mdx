## Introduction

ListItems are flexible interactive elements designed for lists, menus, settings, and option selections. They provide a structured way to display a title with optional description, leading icon, description tag, and various trailing content types.

> View in [Figma](https://www.figma.com/design/JxaLVMTWirCpU0rsbZ30k7/2.-Components-Library?node-id=2255-4155).

## Anatomy

<Canvas of={ListItemStories.WithLeadingIconAndDescription} />

- **Leading Icon (optional)**: Visually represents the item's category or action
- **Title**: The main label of the item (required)
- **Description (optional)**: Provides additional context
- **Description Tag (optional)**: Small tag next to description for status or labels
- **Trailing Content**: Varies based on variant (caret, toggle, value, tag, icon, or none)

### Trailing Content Variants

ListItems support six trailing content variants to suit different interaction patterns:

- **caret**: Navigation indicator (chevron right)
- **toggle**: Switch for on/off states (requires `selected` prop)
- **value**: Displays a primary value with optional subvalue
- **tag**: Shows a tag for status or category
- **icon**: Custom trailing icon
- **none** (default): No trailing content

<Canvas of={ListItemStories.TrailingContentVariantsShowcase} />

### Content Variations

ListItems support various content combinations:

- **Title only**: Basic configuration
- **With description**: Adds context
- **With description tag**: Highlights additional info
- **With leading icon**: Visual identifier
- **With leading icon and description**: Combined for richer items

Note: These can be combined with any trailing variant.

## Responsive Layout

ListItems adapt to their container width, truncating long titles and descriptions as needed.

<Canvas of={ListItemStories.ResponsiveLayout} />

> **Key behaviors:**
>
> - Automatically fits container width
> - Title truncates to one line
> - Description truncates to one line
> - Icons and trailing elements maintain fixed sizes
