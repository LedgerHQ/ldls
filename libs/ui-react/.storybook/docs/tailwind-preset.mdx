# üõ† Tailwind Preset Configuration

The Ledger Design System builds on Tailwind CSS's utility-first methodology while ensuring design consistency through a shared preset. Instead of customizing Tailwind's configuration in every project, **we provide a common preset that all teams can use out of the box. This allows us to:**

- Provide consistent design tokens across all applications
- Enforce a standardized spacing system with precise 1:1 pixel mapping
- Define semantic color tokens that adapt seamlessly to light and dark modes
- Keep typography aligned with our design guidelines

This setup lets teams use the familiar Tailwind syntax while automatically following Ledger's design language.

### üß© Custom Utilities

#### 1. Spacing & Sizing

Our spacing scale uses a 1:1 pixel mapping, where each number directly corresponds to its pixel value. For example, m-4 equals 4px of margin. [Learn more about our spacing system](/docs/foundations-dimensions-spacing--docs).

```tsx
// Spacing Examples
<div className="p-1">        {/* padding: 1px */}
<div className="m-2">        {/* margin: 2px */}
<div className="gap-3">      {/* gap: 3px */}
<div className="p-4">        {/* padding: 4px */}
<div className="m-8">        {/* margin: 8px */}

// Sizing Examples
<div className="w-4">        {/* width: 4px */}
<div className="h-8">        {/* height: 8px */}
<div className="max-w-16">   {/* max-width: 16px */}
```

#### 3. Typography

Our preset carefully **overrides** specific Tailwind core plugins to ensure design consistency.

```typescript
corePlugins: {
  fontSize: false,    // Controlled by our typography system
  fontWeight: false,  // Controlled by our typography system
  lineHeight: false,  // Controlled by our typography system
}
```

Pre-configured typography classes that handle font family, size, weight, line height, and letter spacing. [View typography guidelines](/story/foundations-styles-typography--typography).

```tsx
// Responsive Display Text
<h1 className="responsive-display-1">Large Title</h1>
<h2 className="responsive-display-2">Medium Title</h2>

// Headings
<h2 className="heading-1">Main Heading</h2>
<h3 className="heading-2-semi-bold">Section Title</h3>

// Body Text
<p className="body-1">Regular paragraph text</p>
<p className="body-2-semi-bold">Emphasized text</p>

// ‚ùå These are not valid typography classes
<div className="text-2xl font-semibold">Title</div>
```

## ‚úÖ Best Practices

1. **Use Semantic Color Tokens**

   ```tsx
   // ‚úÖ Do
   <div className="bg-canvas text-base">

   // ‚ùå Don't
   <div className="bg-[#ffffff] text-[#000000]">
   ```

2. **Follow the 4px Grid**

   ```tsx
   // ‚úÖ Do
   <div className="p-4 gap-2">    {/* 4px padding, 2px gap */}

   // ‚ùå Don't
   <div className="p-[5px] gap-[7px]">
   ```

3. **Use Typography System**

   ```tsx
   // ‚úÖ Do
   <h1 className="heading-1">Title</h1>

   // ‚ùå Don't use tailwind classes for typography
   // these classes are not part of our tailwind preset - they are disabled by default.
   <h1 className="text-2xl font-semibold">Title</h1>
   ```

4. **Leverage Dark Mode**

   ```tsx
   // ‚úÖ Do
   <div className="bg-canvas dark:bg-canvas-dark">

   // ‚ùå Don't
   <div style={{
     backgroundColor: isDark ? 'var(--color-canvas-dark)' : 'var(--color-canvas)'
   }}>
   ```
